# What's a commit?

**Sage** wheels their chair over.

> "Forget every git horror story you've heard. At its core, git is *just a
> tool that takes snapshots*. Each snapshot is called a **commit**, and a
> sequence of commits is your project's history.
>
> We'll do the whole loop end-to-end on a tiny folder I left for you at
> `~/work/repo`. Five steps:
>
> 1. Tell git to start tracking the folder (`init`).
> 2. Ask git what it currently sees (`status`).
> 3. Pick which files belong in your next snapshot (`add`).
> 4. Take the snapshot, with a message (`commit`).
> 5. Look at the history you just wrote (`log`).
>
> That's it. Once these click, the rest of git is variations on the same theme."
